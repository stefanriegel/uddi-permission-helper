**js/state.js** — State management module:

Create an ES module that manages per-provider state with NO localStorage (in-memory only per user decision).

```javascript
// State shape (per CONTEXT.md specifics section):
// { aws: { features: {} }, azure: { features: {} }, gcp: { features: {} }, activeProvider: null }

const state = {
  activeProvider: null,
  providers: {
    aws: { features: {} },
    azure: { features: {} },
    gcp: { features: {} }
  }
};
```

Export these functions:
- `getActiveProvider()` — returns `state.activeProvider` (string or null)
- `setActiveProvider(providerId)` — validates providerId is 'aws'|'azure'|'gcp', sets `state.activeProvider`, returns the provider state object
- `getProviderState(providerId)` — returns `state.providers[providerId]` or null if invalid
- `hasProviderData(providerId)` — returns `true` if `Object.keys(state.providers[providerId].features).length > 0` (used by has-data indicator)
- `getState()` — returns the full state object (for debugging/future use)

Validation: `setActiveProvider` should silently ignore invalid provider IDs (return null).

**js/ui.js** — DOM manipulation module:

Import nothing from state.js (receives data as arguments to stay decoupled).

Export these functions:

1. `updateProviderCards(activeProvider, hasDataFn)`:
   - Select all `.provider-card` buttons via `document.querySelectorAll('.provider-card[data-provider]')`
   - For each card, read `card.dataset.provider`
   - Remove `provider-card--selected` and `provider-card--has-data` from all cards
   - Add `provider-card--selected` to the card matching `activeProvider`
   - Set `aria-pressed="true"` on active card, `"false"` on others
   - For non-active cards, call `hasDataFn(providerId)` — if true, add `provider-card--has-data`

2. `updateWorkspace(activeProvider)`:
   - Select `.workspace` section
   - If `activeProvider` is null: add `workspace--hidden`, remove `workspace--active`, set inner HTML to `<p class="workspace__empty">Select a cloud provider above to begin.</p>`
   - If `activeProvider` is set: remove `workspace--hidden`, add `workspace--active`, set inner HTML to `<p class="workspace__provider-label">{displayName} features will appear here</p>` where displayName maps: aws="Amazon Web Services (AWS)", azure="Microsoft Azure", gcp="Google Cloud Platform (GCP)"

Provider display name map (define as const inside ui.js):
```javascript
const PROVIDER_NAMES = {
  aws: 'Amazon Web Services (AWS)',
  azure: 'Microsoft Azure',
  gcp: 'Google Cloud Platform (GCP)'
};
```

**js/app.js** — Entry point:

```javascript
import { setActiveProvider, getActiveProvider, hasProviderData } from './state.js';
import { updateProviderCards, updateWorkspace } from './ui.js';
```

On DOMContentLoaded:
1. Select all `.provider-card[data-provider]` buttons
2. Add click handler to each that:
   a. Reads `button.dataset.provider`
   b. Calls `setActiveProvider(providerId)`
   c. Calls `updateProviderCards(providerId, hasProviderData)`
   d. Calls `updateWorkspace(providerId)`
3. On init, hide workspace: add `workspace--hidden` class to `.workspace` section (so it starts hidden until a provider is clicked — the static HTML shows the empty state text, but once JS loads we hide it and only reveal on selection)

Important: Do NOT re-select the same provider on repeated clicks (if already active, do nothing). Compare with `getActiveProvider()` before proceeding — actually, per the CONTEXT.md there is no mention of toggle behavior, so clicking the already-selected provider should be a no-op.
