**js/ui.js** — Add a new exported function `renderAdvanced(questions, features, onToggle)`:

- `questions` is the same array from getQuestionsForProvider (reuse for grouping)
- `features` is the current features object from state
- `onToggle` is a callback: `(featureId, enabled) => void`

The function renders into `#workspace-content`:

1. For each top-level question, create a category group:
   - Category header: question text as a heading (h3 with class `advanced__category-title`)
   - If the question is standalone (single featureId, no sub-questions):
     - Single checkbox row: checkbox + feature name + permission count badge
     - Checkbox is checked if features[featureId] is true
     - Permission count badge shows number of IAM actions/roles for that feature

   - If the question has sub-questions:
     - Render each sub-question as a checkbox row, indented
     - For exclusive sub-questions (DNS): render as radio inputs sharing same name attribute so browser enforces mutual exclusion. When one is selected, onToggle the selected one to true and the others to false.
     - For non-exclusive sub-questions (GCP Cloud Forwarding): render as independent checkboxes
     - Each checkbox row shows: checkbox/radio + sub-feature display name + permission count badge

2. Permission count badge logic — import the FEATURES objects to get counts:
   - AWS: `feature.actions?.length || feature.policies?.length || 0` — show as "N actions" or "N policies"
   - Azure: Count built-in roles + custom role presence. Show as "N roles" for built-in, or "N permissions" for custom role
   - GCP: `feature.predefinedRoles?.length + feature.customPermissions?.length` — show as "N roles + M permissions" or simplified

   Use a helper function `getPermissionCount(providerId, featureId)` that reads from the appropriate FEATURES object and returns a display string like "21 actions", "3 roles", "10 permissions".

   Import the FEATURES objects in ui.js:
   ```javascript
   import { AWS_FEATURES } from './data/aws.js';
   import { AZURE_FEATURES } from './data/azure.js';
   import { GCP_FEATURES } from './data/gcp.js';
   ```

3. Each checkbox/radio input has `data-feature-id` attribute and a change event listener calling `onToggle`.

Add a `renderCurrentMode()` helper function that:
1. Gets the active provider, current mode, questions, and features
2. If mode is 'wizard': calls `renderWizard(questions, features, handleAnswer)`
3. If mode is 'advanced': calls `renderAdvanced(questions, features, handleToggle)`

`handleAnswer(featureId, enabled)` callback:
1. Calls `setFeature(getActiveProvider(), featureId, enabled)`
2. Calls `renderCurrentMode()` to re-render (wizard state progresses)
3. Calls `updateProviderCards(getActiveProvider(), hasProviderData)` to update has-data indicators

`handleToggle(featureId, enabled)` callback:
1. Same as handleAnswer — calls setFeature, re-renders, updates cards

Wire the mode toggle button:
```javascript
const modeSwitch = document.querySelector('.mode-toggle__switch');
modeSwitch.addEventListener('click', () => {
  const currentMode = getSelectionMode();
  const newMode = currentMode === 'wizard' ? 'advanced' : 'wizard';
  setSelectionMode(newMode);

  // Update toggle visual state
  modeSwitch.setAttribute('aria-checked', newMode === 'advanced' ? 'true' : 'false');

  // Update label active states
  document.querySelectorAll('.mode-toggle__label').forEach(label => {
    label.classList.toggle('mode-toggle__label--active', label.dataset.mode === newMode);
  });

  // Crossfade: fade out, swap content, fade in
  const content = document.getElementById('workspace-content');
  content.classList.add('workspace__content--fading');
  setTimeout(() => {
    renderCurrentMode();
    content.classList.remove('workspace__content--fading');
  }, 200);
});
```
