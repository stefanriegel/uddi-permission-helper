  <behavior>
    **getAzureRoles(selectedFeatureIds)**
    - Returns deduplicated array of { name, builtIn, scope } objects from selected features' roles arrays
    - Empty input returns empty array
    - ['ipamAssetDiscovery'] returns [{ name: 'Reader', builtIn: true, scope: 'subscription' }]
    - ['ipamAssetDiscovery', 'publicDnsReadOnly'] returns [{ name: 'Reader', ... }] (one Reader, NOT two)
    - ['publicDnsReadWrite'] returns [{ name: 'Reader', ... }, { name: 'DNS Zone Contributor', ... }]
    - ['cloudForwardingDiscovery'] returns [] (no built-in roles — only customRole)
    - ['multiSubscription'] returns [] (guidance only)
    - Dedup key is role name string

    **generateAzurePolicy(selectedFeatureIds)**
    - Returns string with Azure CLI `az role assignment create` commands for built-in roles
    - Plus custom role JSON definition block if cloud forwarding features are selected
    - IPAM only: single `az role assignment create --assignee <SERVICE_PRINCIPAL_ID> --role "Reader" --scope /subscriptions/<SUBSCRIPTION_ID>`
    - Cloud Forwarding discovery: custom role JSON with 6 permissions + `az role definition create` command
    - Cloud Forwarding full: custom role JSON with 21 permissions + `az role definition create` command
    - Multi-subscription: guidance comment about management group scope
    - If both IPAM and public DNS read-only selected, Reader appears only ONCE in the az commands

    **generateAzureTerraform(selectedFeatureIds)**
    - Returns azurerm_role_assignment blocks for built-in roles (deduplicated)
    - Plus azurerm_role_definition blocks for custom roles if cloud forwarding selected
    - Uses data sources: azurerm_subscription, azurerm_role_definition
    - Multi-subscription: adds azurerm_management_group data source and management-group-scoped assignments

    **generateAzureGuide(selectedFeatureIds)**
    - Returns numbered step-by-step plain text instructions
    - Starts with service principal creation
    - Lists role assignments needed
    - Ends with "Configure credentials in Infoblox Portal"
    - Multi-subscription adds management group scope instructions
    - Empty selection returns empty string

    **Edge cases:**
    - Empty array returns empty/default for all generators
    - multiSubscription alone returns guidance text only (no roles)
    - Selecting all features deduplicates Reader (appears in 3 features)
  </behavior>

`getAzureRoles(selectedFeatureIds)`:

`generateAzurePolicy(selectedFeatureIds)`:
- Get deduplicated roles via getAzureRoles
- For each role: output `az role assignment create --assignee "<SERVICE_PRINCIPAL_ID>" --role "{roleName}" --scope "/subscriptions/<SUBSCRIPTION_ID>"`
- For cloud forwarding features: output custom role JSON with `az role definition create --role-definition '{...}'`
- For multi-subscription: append guidance comment about management group scope
- Return combined string

`generateAzureTerraform(selectedFeatureIds)`:
- Get deduplicated roles
- For each unique built-in role: output azurerm_role_assignment block with data.azurerm_role_definition lookup
- For cloud forwarding custom roles: output azurerm_role_definition + azurerm_role_assignment
- For multi-subscription: output management group scoped variant
- Start with `data "azurerm_subscription" "current" {}` and `data "azurerm_client_config" "current" {}`
- Return combined HCL string

`generateAzureGuide(selectedFeatureIds)`:
- Numbered plain text steps
- Step 1: Create service principal (or use existing)
- Middle steps: Role assignments for each selected feature
- Last step: Configure credentials in Infoblox Portal
- Multi-subscription: Add management group scope instructions
- Return joined string
