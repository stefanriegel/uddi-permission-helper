**js/output.js — Add policy size warning logic:**

1. Define constants at module top:
   ```
   const AWS_POLICY_SIZE_LIMIT = 6144;
   const AWS_POLICY_SIZE_WARN_THRESHOLD = 0.8; // 80% = 4915 chars
   ```

2. Create a new function `renderPolicySizeWarning(providerId, policyText)`:
   - Only applies when `providerId === 'aws'`
   - For non-AWS providers, remove any existing warning element and return
   - Calculate `policyText.length` (the raw policy text, NOT the annotated version with // comments — use `generateAwsPolicy(selectedIds)` output since that is the actual IAM policy JSON that AWS evaluates)
   - If length >= `AWS_POLICY_SIZE_LIMIT * AWS_POLICY_SIZE_WARN_THRESHOLD` (4915):
     - Find or create a `div.output__size-warning` element
     - Insert it as the first child of `.output__content`
     - Set innerHTML to: warning icon (Unicode `\u26A0`) + text: `"Policy size: {length} / 6,144 characters ({percent}%). AWS managed policies cannot exceed 6,144 characters."`
     - If length >= 6144, add class `output__size-warning--exceeded` and change text to: `"Policy size: {length} / 6,144 characters — EXCEEDS AWS managed policy limit!"`
   - If length < threshold, remove warning element if it exists

3. Call `renderPolicySizeWarning(providerId, rawPolicy)` inside `renderOutput()` right after the provider content generation block (after the if/else if chain). Compute `rawPolicy` by calling `generateAwsPolicy(selectedIds)` when providerId is 'aws', empty string otherwise.

**index.html — Add cache-control meta hints for offline support:**

Add these two `<meta>` tags in `<head>` after the existing `<meta name="description">` tag:
```html
<meta http-equiv="Cache-Control" content="public, max-age=86400">
<meta http-equiv="Expires" content="86400">
```

These are advisory hints for browsers that respect HTTP-equiv meta tags. The real cache headers come from the hosting provider, but these provide a baseline.
