  <action>
Add four exported functions to `js/data/aws.js`:

**1. `getAwsActions(selectedFeatureIds)`**
- Takes array of feature ID strings (e.g., `['vpcIpamDiscovery', 'ec2Networking']`)
- For each selected feature that has an `actions` array, collect all actions
- Deduplicate using `[...new Set(allActions)]`
- Sort alphabetically for consistent output
- Return the sorted unique array
- Skip features that don't have `actions` (like multiAccount which has `policies`)

**2. `generateAwsPolicy(selectedFeatureIds)`**
- Call `getAwsActions(selectedFeatureIds)` to get deduplicated actions
- Build IAM policy JSON document:
```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "InfobloxUDDIPermissions",
      "Effect": "Allow",
      "Action": [...deduplicated actions...],
      "Resource": "*"
    }
  ]
}
```
- If `multiAccount` is in selectedFeatureIds, append its three policy documents as additional sections (trust policy, organizations reference, STS policy)
- Return `JSON.stringify(policy, null, 2)` for pretty-printed output
- Include inline comment rationale for each action using the rationale data from AWS_FEATURES

**3. `generateAwsTerraform(selectedFeatureIds)`**
- Get deduplicated actions via `getAwsActions(selectedFeatureIds)`
- Generate a single combined `aws_iam_policy` resource with all actions in a `jsonencode` policy document
- If multiAccount is selected, also generate `aws_iam_role` with assume_role_policy and `aws_iam_role_policy_attachment` for Organizations
- Return the complete HCL string

**4. `generateAwsGuide(selectedFeatureIds)`**
- Compose a numbered step-by-step guide combining relevant setup instructions
- Standard features: "1. Open IAM Console > Policies > Create Policy. 2. Paste the JSON policy below. 3. Name the policy 'InfobloxUDDI-Discovery'. 4. Create IAM role 'InfobloxUDDI-DiscoveryRole'. 5. Attach the policy to the role. 6. Configure the role ARN in Infoblox Portal."
- If multiAccount is selected, add sub-account role creation steps, trust policy configuration, External ID instructions
- Return the guide as a plain text string with numbered steps

All four functions must be pure (no side effects, no DOM access) — they take feature IDs and return strings.
