  <name>Task 1: Create output.js rendering module</name>
  <files>js/output.js</files>
  <action>
Create `js/output.js` as an ES module with these exports:

**`renderOutput(providerId, features)`** — Main rendering function:
1. Extract selected feature IDs: `Object.keys(features).filter(id => features[id] === true)`
2. If no features selected, set all three panels to placeholder text "Select features to generate your permission policy." in `--color-output-muted` and return.
3. Dispatch to provider-specific generators based on `providerId`:
   - `aws`: call `generateAwsPolicy`, `generateAwsTerraform`, `generateAwsGuide`
   - `azure`: call `generateAzurePolicy`, `generateAzureTerraform`, `generateAzureGuide`
   - `gcp`: call `generateGcpPolicy`, `generateGcpTerraform`, `generateGcpGuide`
4. For the Policy tab content, inject rationale comments (per OUT-07):
   - AWS: Before setting innerHTML, enhance the JSON output by inserting `// {rationale}` comments. Parse the policy JSON, and for each action in the Action array, look up the rationale from the selected features' `.rationale` objects and produce a formatted display: wrap in `<pre><code class="language-json">` and add rationale as `// comment` lines above each action group.
   - Azure: The generateAzurePolicy output is Azure CLI commands. Add rationale as `# {rationale}` comment lines before each `az role assignment create` and `az role definition create` command. Collect rationale from selected features' `.rationale` objects. Wrap in `<pre><code class="language-bash">`.
   - GCP: The generateGcpPolicy output is gcloud commands. Add rationale as `# {rationale}` comment lines before each `gcloud` command. Wrap in `<pre><code class="language-bash">`.
5. For the Terraform tab: wrap content in `<pre><code class="language-hcl">` tags.
6. For the Setup Guide tab: wrap content in `<pre><code>` tags (no language class — plain text).
7. Set `innerHTML` of `#panel-policy`, `#panel-terraform`, `#panel-guide` respectively.
8. After setting innerHTML, call `Prism.highlightAllUnder(outputSection)` if `window.Prism` is available (graceful degradation if CDN not loaded).

**`updateBadge(providerId, features)`** — Badge counter (per OUT-06):
1. Extract selected feature IDs same as above.
2. Calculate permission count per provider:
   - AWS: `getAwsActions(selectedIds).length` actions + count policies for multiAccount if selected (3 policies)
   - Azure: `getAzureRoles(selectedIds).length` roles + sum of custom role permission counts for selected features with `.customRole`
   - GCP: `getGcpRoles(selectedIds).length` roles + `getGcpCustomPermissions(selectedIds).length` permissions
3. Build display text: `"{count} permissions"` (use "permissions" as universal term for badge)
4. Update `.output__badge` text content.
5. Toggle CSS class `output__badge--active` when count > 0, remove when count === 0.

**Rationale injection strategy for AWS Policy tab:**
The `generateAwsPolicy` returns a JSON string. Instead of parsing and re-serializing, build a custom annotated output:
- Parse the generated JSON to get the actions array
- For each action, look up rationale from ALL selected features' `.rationale` objects (first match wins since action string is unique)
- Build output like:
```
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "InfobloxUDDIPermissions",
      "Effect": "Allow",
      "Action": [
        // List VPCs for network topology discovery
        "ec2:DescribeVpcs",
        // Enumerate subnets within each VPC
        "ec2:DescribeSubnets",
        ...
      ],
      "Resource": "*"
    }
  ]
}
```
Use a dedicated `buildAnnotatedAwsPolicy(selectedIds)` helper for this.

For Azure and GCP policy tabs, the rationale is simpler — insert `# comment` lines before each command in the CLI output.

**Empty state handling:** When `selectedIds.length === 0`, show the placeholder message in muted color and disable badge (show "0 permissions").

   ```javascript
   const providerId = getActiveProvider();

   if (providerId && features) {
     renderOutput(providerId, features);
     updateBadge(providerId, features);
   }
   ```
3. Call `refreshOutput()` at the end of `handleAnswer()` (after `renderCurrentMode()`)
4. Call `refreshOutput()` at the end of the provider card click handler (after `renderCurrentMode()`)
5. Call `refreshOutput()` at the end of the mode switch handler — the output should reflect current state regardless of mode.
